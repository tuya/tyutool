
# Batch Auth 批量授权使用指南

本文说明 tyuTool 图形界面中 **Batch Auth** 标签页的能力：通过串口写入 TuyaOpen 授权信息，并维护授权 Excel 表。使用前请一并阅读仓库根目录下的 [Terms_And_Agreements.md](../../Terms_And_Agreements.md)。

---

## 重要提示（开始之前）

**在点击「开始授权」或选择授权表进行批量任务前，请务必自行将授权 Excel（`.xlsx`）完整备份到安全位置**（例如复制到其他目录、纳入版本管理或团队约定的网盘）。授权成功后会**直接改写**该工作簿中的行（状态、MAC、时间戳等）。程序虽可能在同目录下生成 `原文件名.xlsx.bak`，或在你首次确认时尝试复制一份备份，但**不能替代**你主动、可恢复的备份策略。产线或批量任务**开始前**请再次确认备份可用。

**本工具按设备上报的 MAC 与 Excel 中的记录做关联与决策；工具不会替你校验「不同模组之间的 MAC 是否重复」。请你在产线与物料层面自行保证：每一台模组内的 MAC 在业务范围内是唯一的、不与其他设备冲突。** 若存在重复 MAC，可能导致授权记录错乱或与预期不符。

英文版说明见：[Batch-Auth-Guide.md](../en/Batch-Auth-Guide.md)。

## 第一部分：Batch Auth 实现流程

本节从实现角度说明工具内部如何串联 **界面、后台线程、串口协议与 Excel**，便于理解现象与日志；不要求终端用户阅读代码。

### 1. 整体结构

- 主界面由 **Batch Auth** 页与后台 **工作线程**（`AuthWorker`）配合：点击 **Start** 后，授权与烧录在子线程执行，避免阻塞界面。
- 核心逻辑在 **AuthHandler**：负责打开串口、与设备交互、读写授权、驱动 Excel 更新；**AuthExcelParser** 负责解析与保存 `.xlsx`。
- 可选 **先烧录固件**：当用户同时指定了芯片与固件文件时，线程内先走烧录与复位，再进入与「仅授权」相同的串口授权流程。

### 2. 设备固件 CLI：auth、auth-read、read_mac（必备）

Batch Auth 在 PC 端通过 **串口**与设备上的 **TAL CLI** 交互，**依赖固件侧已注册且可用的三条命令**：**`auth`**、**`auth-read`**、**`read_mac`**。名称与交互语义须与 tyuTool 内置协议一致；若未启用或未实现等效命令，**开始授权**与 **Read MAC** 将无法完成。

在 **TuyaOpen** 公开仓库中，可按以下路径对照集成（路径均相对于该仓库根目录）：

- **`apps/tuya_cloud/switch_demo/src/tuya_main.c`**：在 `user_main` 中，`tal_kv_init` 之后依次调用 **`tal_cli_init`**、**`tuya_authorize_init`**、**`tuya_app_cli_init`**，用于启动 CLI、注册授权子命令以及应用层命令表。若目标平台为 Linux 桌面等非嵌入式环境，部分初始化可能被条件编译跳过，以你方实际工程为准。
- **`src/tuya_cloud_service/authorize/tuya_authorize.c`**：**`tuya_authorize_init`** 向 CLI 注册 **`auth`**（按固定长度将 UUID、AuthKey 写入 KV）、**`auth-read`**（从 KV 读出当前授权）、以及 **`auth-reset`**（按需清除 KV 中的授权项）等；KV 键名例如 **`UUID_TUYAOPEN`**、**`AUTHKEY_TUYAOPEN`**。Batch Auth 主要依赖 **`auth`** 与 **`auth-read`** 与 Excel/设备状态对齐。
- **`apps/tuya_cloud/switch_demo/src/cli_cmd.c`**：**`tuya_app_cli_init`** 将 **`read_mac`** 与演示用其它命令一并注册到 **`s_cli_cmd`**。参考实现里 **`read_mac`** 常与 WiFi 相关宏一起编译；若你的芯片或网络方案不同，**仍需提供与协议兼容的 `read_mac` 实现**（例如从本机网络栈读取 MAC 并输出），保证 PC 端解析到的 MAC 格式符合预期。

**若你希望在一段时间之后使上述 CLI 命令（或整条串口调试/授权通道）失效**，**须在你方固件中自行实现**，常见做法仅包括：**上电计时或累计运行时间到达阈值后不再注册相关命令**，或 **在发布镜像中不调用 `tal_cli_init` / 授权模块初始化**。**tyuTool 不提供「到期自动关闭 CLI」一类能力**。

集成时请将上述调用与命令注册纳入你方应用启动流程；仅烧录固件而未打开对应 CLI 与三条命令时，Batch Auth 仍会失败。

### 3. 会话日志

- 每次授权会话会在 **授权 Excel 所在目录** 生成独立日志文件，文件名形如：`表名_auth_时间戳.log`，用于排障与追溯（界面中可复制完整路径）。
- **建议每次操作结束后，将当次及相关日志另行保留、归档**（例如按日期或批次拷贝到独立目录），便于日后问题复现与排查。

### 4. 授权 Excel 的处理

- 启动授权时加载工作簿，解析表头中的 **UUID**、**AUTHKEY**（或列名 **key**）等列；若缺少 **STATUS / MAC / TIMESTAMP**，在需要写入磁盘时自动追加列。
- 对同一文件使用 **文件锁**（`.lock`），避免多实例同时改写；统计 **总数 / 已用 / 剩余** 未用行。
- 写入设备并校验成功后，将对应行标记为 **USED**，并写入 **MAC** 与 **时间戳**，然后保存文件；首次保存前可能对原表做 **`.bak` 备份**（若尚不存在）。

### 5. 串口侧与可选烧录

- **未选择固件**：打开串口 → 排空上电启动阶段的串口数据以便进入可交互状态 → 进入读 MAC 与授权流程。
- **已选择固件（当前为 ESP32）**：先校验串口可用 → 按芯片调用既有 **烧录流程**（握手、擦除、写入、校验、重启）→ 通过 **DTR/RTS** 复位设备 → 再进入与上一条相同的读 MAC、授权与校验流程（串口会再次按授权波特率打开）。

### 6. 单台设备授权逻辑（摘要）

流程默认 **每台物理设备有唯一 MAC**；若多台模组 MAC 相同，表内按 MAC 匹配与写入的逻辑可能无法区分真实设备，**需由你在硬件/烧录侧保证 MAC 不重复**。

1. **读 MAC**（失败会按上限重试）。
2. **读设备当前授权**：若设备里是占位或未授权，则按表分配新行；若已授权，则与 Excel 中 **按 MAC 匹配的行** 比对。
3. **决策**：若与表中记录一致可跳过写入；若不一致或 MAC 不在表中，可能通过对话框询问是否 **覆盖**（用户可 **Copy** 提示内容）。
4. **写入与校验**：从表中取 **下一条未使用** 的 UUID/AUTHKEY（或复用某行），经协议 **写入** 设备，再 **回读比对**；全部通过后才 **回写 Excel** 标记已用。

### 7. 界面步骤条

- 界面上的步骤（打开串口、烧录、复位、读 MAC、写入授权、校验、关闭串口等）与上述阶段对应，用于展示当前进度与失败步骤。

## 第二部分：使用操作指南

### 1. 环境准备

1. 运行 **tyuTool GUI**（`tyutool_gui.py` 或已打包的可执行文件）。
2. USB 转串口已连接设备且驱动正常；Linux 上常见为 `/dev/ttyUSB*`、`/dev/ttyACM*` 等。
3. 准备好业务侧提供的 **授权表 .xlsx**，并**已完成本文开头所述的自行备份**。
4. **在模组与产线流程上确保各设备 MAC 唯一、不重复**（见文首「重要提示」）。
5. **设备固件已启用 TAL CLI，并注册 `auth`、`auth-read`、`read_mac`**（见第一部分第 2 节）。
6. 处理 Excel 依赖 **openpyxl**（见项目 `requirements.txt`）。

### 2. 授权 Excel 格式

- **第一行为表头**，列名不区分大小写，支持别名：
  - **必填**：`UUID`；`AUTHKEY` 或 **`key`**（二者都表示授权密钥列）。
  - **可选**：若已有 `STATUS`、`MAC`、`TIMESTAMP` 列则使用之；若无，程序在首次需要写入时会 **自动追加** 这些列。
- **统计**：自第 2 行起，**UUID 非空**计为一条；`STATUS` 为 `USED`（不区分大小写）计为已使用。
- **分配**：自上而下取 **第一条** UUID、AUTHKEY 均有值且 **未标记 USED** 的记录作为本次写入的授权。

### 3. 界面操作步骤

#### 3.1 串口与波特率

- **Port**：点击 **Rescan** 刷新后选择设备串口。
- **Baud**：常见 `115200`，也可选 `230400` / `460800` / `921600`（需与设备一致）。

#### 3.2 选择授权表

- 通过 **Browse** 选择 `.xlsx`。
- 加载成功后显示 **Total / Used / Remain**。
- 界面会提示授权将 **修改 Excel**；**请再次确认你已自行备份**（见文首「重要提示」）。确认后若同目录尚无 `.bak`，程序可能尝试生成 **`原文件名.xlsx.bak`**。

#### 3.3（可选）先烧录再授权

- **Chip** 选 **ESP32**，**Firmware** 选择 `.bin` 文件。
- 不选固件时，流程不包含烧录与 DTR/RTS 复位，仅串口授权。

#### 3.4 开始授权

- 已选 **Port** 且已选 **Excel** 后，**Start** 可用。
- 运行中显示步骤列表与进度；日志路径可点击复制（见第一部分第 3 节「会话日志」）。

#### 3.5 仅读 MAC

- 选好 **Port** 与 **Baud** 后点击 **Read MAC**，仅读取 MAC，不写授权、不改表。

### 4. 设备固件：KV 密钥与数据安全（防泄露）

若不希望模组上的 **KV 存储数据**（包含但不限于授权相关信息）因使用公开、默认或与竞品相同的密钥而被他人按同样参数解密、读取，应在 **设备侧固件** 中自行修改 **`tal_kv_init`** 的配置：将 **`seed`**、**`key`** 替换为 **仅你方掌握** 的随机串，并 **妥善保管、切勿泄露**。KV 中的数据通常依赖该 **`seed` / `key` 参与派生或加解密**；密钥一旦泄露，对应 KV 内的敏感内容（含授权信息）将面临被还原的风险。

请在工程中将初始化修改为 **你方自有** 的密钥，例如：

```c
tal_kv_init(&(tal_kv_cfg_t){
    .seed = "vmlkasdh93dlvlcy",
    .key  = "dflfuap134ddlduq",
});
```

**说明**：上式仅示意字段结构；**务必将 `seed`、`key` 改为你方自行生成且保密的值**，不得长期沿用文档、示例或 SDK 中的公开默认值。生产与版本发布流程中需防止密钥进入公开仓库、日志或外发文档。

### 5. 常见问题

- **Remain 为 0**：无可用行，需补充授权或检查是否均已 USED。
- **读 MAC 失败**：检查接线、波特率、固件是否进入可交互状态；确认已启用 CLI 且 **`read_mac`** 可用（见第一部分第 2 节）；可先烧录支持授权协议的固件。
- **Excel 报错**：确认首行包含 **UUID** 与 **AUTHKEY**（或 **key**）。
- **无法加锁 / 提示文件被占用**：关闭其他正在使用该 `.xlsx` 的 tyuTool 实例。
